# Gopher

**论文**：*Scaling Language Models: Methods, Analysis & Insights from Training Gopher*（Rae et al., 2021）
**arxiv**：2112.11446
**机构**：DeepMind

## 核心问题

系统研究语言模型从 4400 万到 2800 亿参数的 scaling 规律，以及数据质量、训练方法对 152 项任务性能的影响。

## 数据处理 Pipeline（MassiveText）

这是论文最值得关注的部分，Llama 3 明确引用了其中的过滤方法。

**数据集构成**（共 2.35 亿文档，约 10.5 TB）：

| 来源 | 占比 | 训练采样比 |
|------|------|-----------|
| MassiveWeb（网页） | 48% | 50% |
| Books | 27% | 30% |
| C4 | 10% | 10% |
| News | 10% | 10% |
| GitHub | 3% | — |
| Wikipedia | 2% | — |

**质量过滤规则**（启发式，不用分类器）：

- 文档长度：50–100,000 词
- 平均词长：3–10 字符
- 符号字符比 < 0.1
- 含字母字符比 > 80%
- 超过 90% 的行不以符号开头或结尾

**重复 n-gram 覆盖率过滤**（Llama 3 引用的核心方法）：

检测文档内部的重复内容，用于过滤日志、错误信息等"行级去重检测不到的重复"——这类内容每行都是唯一的（所以行级去重无效），但行内有大量重复片段。

阈值设置：

| 指标 | 阈值 |
|------|------|
| 重复行分数 | > 0.30 |
| 重复段落分数 | > 0.30 |
| 2-gram 重复字符分数 | > 0.20 |
| 3–4-gram 重复字符分数 | > 0.18 |
| 5–10-gram 重复字符分数 | > 0.10 |

**文档去重**：MinHash 计算 13-gram Jaccard 相似度，相似度 > 0.8 判为近重复，随机删除其一。

**测试集污染过滤**：移除与 WikiText103、C4 等测试集相似度 > 0.8 的训练文档。

## 模型规模和训练

6 个规模：44M、117M、417M、1.4B、7.1B、**280B**（旗舰）。

280B 模型：80 层，128 头注意力，训练 300B token，上下文 2048 token，bfloat16 混合精度。

## 关键结果

- 152 项任务中 81% 超越此前 SOTA（GPT-3、Jurassic-1、Megatron-NLG）
- 阅读理解显著提升：RACE-h 71.6% vs GPT-3 47.9%
- 数学、逻辑推理任务收益有限（这推动了后续 chain-of-thought 等研究方向）
- 发现：**更大的模型在被提示有毒内容时生成毒性更强**，但对话提示可以抑制

## 和后续工作的关系

- **重复 n-gram 覆盖率**被 Llama 3 直接引用，用于过滤行级去重无法处理的重复内容
- 数据质量启发式过滤的系统化做法影响了 C4、The Pile、ccNet 等后续数据集
- 模型规模分析为 Chinchilla（2022）的 optimal compute 研究奠定了基础

## 和 ccNet 的区别

| | ccNet | Gopher/MassiveText |
|--|-------|-------------------|
| 质量过滤 | KenLM 困惑度（模型打分） | 启发式规则（不用模型） |
| 去重 | MinHash + 行级哈希 | MinHash + 重复 n-gram |
| 多语言 | 100+ 语言 | 主要英文 |
| 开放程度 | 开源代码和数据 | 仅论文，数据未公开 |
