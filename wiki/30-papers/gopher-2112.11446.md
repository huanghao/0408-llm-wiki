# Gopher

**论文**：*Scaling Language Models: Methods, Analysis & Insights from Training Gopher*（Rae et al., 2021）
**arxiv**：2112.11446
**机构**：DeepMind

## 核心问题

系统研究语言模型从 4400 万到 2800 亿参数的 scaling 规律，以及数据质量、训练方法对 152 项任务性能的影响。

## 数据处理 Pipeline（MassiveText）

这是论文最值得关注的部分，Llama 3 明确引用了其中的过滤方法。

**数据集构成**（共 2.35 亿文档，约 10.5 TB）：

| 来源 | 占比 | 训练采样比 |
|------|------|-----------|
| MassiveWeb（网页） | 48% | 50% |
| Books | 27% | 30% |
| C4 | 10% | 10% |
| News | 10% | 10% |
| GitHub | 3% | — |
| Wikipedia | 2% | — |

**质量过滤规则**（启发式，即人工设计的规则，不用机器学习分类器）：

- 文档长度：50–100,000 词
- 平均词长：3–10 字符
- 符号字符比 < 0.1
- 含字母字符比 > 80%
- 超过 90% 的行不以符号开头或结尾

**重复 n-gram 覆盖率过滤**（Llama 3 引用的核心方法）：

**定义**：计算文档里被重复 n-gram 覆盖的字符比例。比如一段文字里 30% 的字符都属于在文档内重复出现过的 2-gram，说明这段文字有大量内部重复，信息密度极低。这不是跨文档的重复（行级去重处理那个），而是**单篇文档内部**的重复——每行可能都是唯一的，但行内充满重复片段（如 `error: error: error: ...` 或日志中反复出现的模板短语）。

**和信息量的关系**：你说得对。从文本层面看这些行是"不重复的"（每行不同），但从信息量角度看它们对模型训练没有价值——模型只需要见过一次这种模式就够了，反复见到只是浪费。

阈值设置（超过阈值的文档被删除）：

| 指标 | 阈值 | 含义 |
|------|------|------|
| 重复行分数 | > 0.30 | 超过 30% 的行在文档内重复出现 |
| 重复段落分数 | > 0.30 | 超过 30% 的段落在文档内重复出现 |
| 2-gram 重复字符分数 | > 0.20 | 超过 20% 的字符属于重复 2-gram |
| 3–4-gram 重复字符分数 | > 0.18 | 超过 18% 的字符属于重复 3/4-gram |
| 5–10-gram 重复字符分数 | > 0.10 | 超过 10% 的字符属于重复 5–10-gram |

阈值随 n-gram 长度递减：越长的 n-gram 在正常文本里本来就很少重复，一旦重复说明问题更严重，所以容忍度更低。

**文档去重**：MinHash 计算 **13-gram** Jaccard 相似度，相似度 > 0.8 判为近重复，随机删除其一。

这里用 13-gram（词级）而不是 5-gram，是因为两者目的不同：
- 质量过滤里的 n-gram 是**字符级**，粒度小，用来检测行内重复片段
- 去重里的 13-gram 是**词级**，粒度大，用来检测两篇文档是否"整体上说的是同一件事"

13 个词大约是 1–2 句话的长度，这个粒度下两篇文档有大量相同的 13-gram，基本可以确定是近重复（转载、复制粘贴）。用 5-gram 粒度太小，很多不相关的文档也会碰巧共享大量 5-gram（比如都包含"the quick brown"这种常见短语），导致误判。

**测试集污染过滤**：移除与 WikiText103、C4 等测试集相似度 > 0.8 的训练文档。

## 模型规模和训练

6 个规模：44M、117M、417M、1.4B、7.1B、**280B**（旗舰）。

280B 模型：80 层，128 头注意力，训练 300B token，上下文 2048 token，bfloat16 混合精度。

## 关键结果

- 152 项任务中 81% 超越此前 SOTA（GPT-3、Jurassic-1、Megatron-NLG）
- 阅读理解显著提升：RACE-h 27.2%（44M）→ 71.6%（280B），GPT-3 为 47.9%
- 数学、逻辑推理任务收益有限（这推动了后续 chain-of-thought 等研究方向）
- 发现：**更大的模型在被提示有毒内容时生成毒性更强**，但对话提示可以抑制

**这些提升是数据质量还是模型规模带来的**：

论文明确表示"scale and the dataset are both contributing"——两者都有贡献，无法完全拆分。但有几个证据说明数据质量独立起作用：

- 论文指出，一个 223M 参数的 ALBERT-XXL（在 RACE 上 fine-tune 过）仍能超过 Gopher 280B，说明模型规模不是唯一决定因素，任务相关数据质量也很关键
- Appendix A.3 有数据质量[消融实验](../20-concepts/ablation-study.md)，显示去掉质量过滤后性能下降
- MassiveText 经过质量过滤后，10.5 TB 的数据最终只用了其中一部分训练（训练 300B token，约占总数据量的 30%），说明过滤是有实质性筛选作用的

**无法精确拆分的原因**：数据质量和规模是耦合的——更大的模型需要更多数据，更多数据里质量参差不齐，所以两者不是独立变量，很难做到完全控制一个变量改变另一个。DCLM（2024）后来做了更系统的对照实验（固定模型只改数据），是这个方向更好的参考。

## 和后续工作的关系

- **重复 n-gram 覆盖率**被 Llama 3 直接引用，用于过滤行级去重无法处理的重复内容
- 数据质量启发式过滤的系统化做法影响了 **C4**、**The Pile**、ccNet 等后续数据集

  **C4**（Raffel et al., 2020，Google）：T5 模型的训练数据集，从 Common Crawl 里清洗出约 750 GB 的英文文本。过滤规则和 Gopher 类似：去掉过短的行、含脏词的页面、代码页面等。C4 是 BERT/T5 时代最常用的预训练数据集之一，"脏词计数"过滤方法后来被 Llama 3 引用。

  **The Pile**（Gao et al., 2020，EleutherAI）：825 GB 的多源数据集，包含 22 个子集（Books、GitHub、ArXiv、Wikipedia、Stack Exchange 等），是第一个系统整合多种高质量来源的开源预训练数据集，影响了后续 The Pile 系列和 RedPajama 等。
- 模型规模分析为 Chinchilla（2022）的 optimal compute 研究奠定了基础

## 和 ccNet 的区别

| | ccNet | Gopher/MassiveText |
|--|-------|-------------------|
| 质量过滤 | KenLM 困惑度（模型打分） | 启发式规则（不用模型） |
| 去重 | MinHash + 行级哈希 | MinHash + 重复 n-gram |
| 多语言 | 100+ 语言 | 主要英文 |
| 开放程度 | 开源代码和数据 | 仅论文，数据未公开 |

## Section 4：Scaling 分析

**哪些任务受益最大**（规模从 44M 扩大到 280B 时）：
- 医学、科学、技术、人文社科类知识问答：提升显著，280B 在很多子任务上接近或超过人类水平
- 阅读理解（RACE-h）：27.2% → 71.6%，提升 44 个百分点
- 事实核查（FEVER）：超越有监督 SOTA
- 词语类比、修辞识别：大幅提升

**哪些任务收益有限**：
- 数学推理（Abstract Algebra、High School Mathematics）：280B 反而比 7.1B 更差
- 逻辑推理（Temporal Sequences）：规模带来的提升微乎其微
- 常识推理的某些子类

**为什么有差异**：知识密集型任务（需要记住事实）随规模提升明显；需要多步推理的任务不行——模型参数多了，但"思考步骤"还是一次前向传播，没有额外的计算空间。这个发现直接推动了 chain-of-thought prompting（让模型先写推理步骤再给答案）的研究。

## Section 6：毒性与偏见分析

### 毒性（Toxicity）

**毒性**指模型生成的内容中包含仇恨言论、侮辱、威胁、色情、歧视等有害内容。论文用 Perspective API（Google 的文本毒性评分工具）自动给模型输出打分，分数 0–1，越高越有毒。

**有毒 prompt 是什么**：研究者从网上收集了一批本身含有仇恨或攻击性语言的文本，作为 prompt 输入给模型，让模型续写。例如：
- 含种族歧视语言的句子，让模型接着写
- 含攻击性语言的开头，让模型补完

**两种实验设置的结果**：

- **无提示（unprompted）**：让模型自由生成文本，不给任何有害引导。不同规模的模型生成有毒内容的比例和训练数据的毒性水平相当，规模增大没有显著变化。
- **有毒提示（toxic prompt）**：给模型一个有毒的开头，要求续写。更大的模型生成的毒性内容更强——因为大模型更善于"理解并延续 prompt 的风格"，包括有害风格。这个效应在 7.1B 附近趋于饱和。
- **对话引导**：在 prompt 里加入"请礼貌地回答"等指令，大模型能有效降低毒性——说明模型的毒性输出是可以被引导控制的。

**评估方法**：Perspective API 对每段生成文本打毒性分数，统计超过阈值（如 0.5）的比例。

### 偏见（Bias）

**偏见**指模型对不同群体（性别、种族、方言）的处理方式存在系统性差异。

**职业性别刻板印象**：给模型"一位护士"或"一位工程师"，看模型接下来倾向于用"她"还是"他"。例如：
- 模型在"护士是___"后更倾向于填"她"（符合刻板印象）
- 在"工程师是___"后更倾向于填"他"
- 这种偏差在所有规模下都存在，规模增大不会消除

**Winogender 测试**：一组专门设计的代词消歧题，分"顺刻板印象"和"逆刻板印象"两类：
- 顺刻板印象例子："护士通知医生，**她**需要签字。"（护士=女性，符合刻板印象）
- 逆刻板印象例子："护士通知医生，**他**需要签字。"（护士=男性，违反刻板印象）
- 模型在顺刻板印象句子上准确率更高，说明它学到了社会偏见

**方言偏见**：模型对非裔美国英语（AAVE）的困惑度（perplexity）高于标准美式英语，说明模型把 AAVE 视为"更不正常"的文本，且这个差距随规模增大不收窄。

**结论**：规模扩大可以提升能力，但不能解决毒性和偏见问题，需要专门的对齐技术（这正是 RLHF、Constitutional AI 等后续工作的动机，详见 [RLHF](../20-concepts/rlhf.md)）。

## 现状与影响

**定性：奠基性工作，数据处理方法仍被引用，模型本身已被超越。**

Gopher 模型（280B，2021 年底）早已不是 SOTA，DeepMind 内部后续也转向 Chinchilla 路线（更小模型 + 更多数据）。但这篇论文的**数据处理方法**至今有实际影响：重复 n-gram 覆盖率过滤被 Llama 3 直接引用，MassiveText 的启发式过滤规则影响了 C4、The Pile 等主流数据集。

如果你在做预训练数据清洗，Gopher 的 Appendix A 是必读参考；如果你关心的是模型能力，读 Chinchilla（2022）更相关。
