# nuScenes

**论文**：*nuScenes: A multimodal dataset for autonomous driving*（Caesar et al., 2020）
**arxiv**：1903.11027
**机构**：Motional（原 nuTonomy）/ nuScenes team
**发表**：CVPR 2020

## 核心问题

**如何构建一个覆盖完整驾驶场景、传感器齐全、标注丰富的公开自动驾驶数据集，使研究者能够在统一基准上评估感知算法？**

nuScenes 之前，主流数据集（KITTI）只有单目相机 + 稀疏 LiDAR，只标注了前向视角，且规模小。nuScenes 的目标是提供 360° 全向感知、多传感器融合、完整标注的大规模基准。

## 数据集规模

| 指标 | 数值 |
|------|------|
| 场景数（splits） | 1000 个场景（700 训练 / 150 验证 / 150 测试） |
| 每个场景时长 | 20 秒 |
| 关键帧（keyframe）数 | 40,000 帧（每 0.5 秒一帧，2Hz） |
| 3D 标注框总数 | 1,400,000+ |
| 标注类别数 | 23 个（合并为 10 个评估类别） |
| 采集城市 | 波士顿 + 新加坡（2 个城市，4 个地图区域） |

## 传感器配置

这是 nuScenes 的核心设计决策：**完整的量产级传感器套件**，不只是研究用的稀疏配置。

```
传感器配置（安装在车顶）：

  1× 旋转式 LiDAR（Velodyne HDL-32E）
    - 32 线，360° 水平视角
    - 20Hz 扫描频率
    - 有效范围：~70m

  6× 相机（覆盖 360° 全向）
    - 前方（0°）、前左（-55°）、前右（+55°）
    - 后方（180°）、后左（-110°）、后右（+110°）
    - 分辨率：1600×900，12Hz

  5× 毫米波雷达（Continental ARS 408-21）
    - 前方 1 个 + 前左/前右/后左/后右各 1 个
    - 远程模式：~250m，±9° 视角
    - 优势：对速度直接测量，不受天气影响

  1× GPS / IMU（定位和同步）
```

**为什么配置 5 个雷达**：雷达对速度敏感（多普勒效应），可以直接测量目标速度，而 LiDAR 只能通过相邻帧差分估计速度。5 个雷达实现全向速度感知，对于运动预测（MOT）任务至关重要。

**相机和 LiDAR 的频率差异**：LiDAR 20Hz，相机 12Hz，关键帧 2Hz——这是 nuScenes 数据处理 pipeline 的核心设计：只在关键帧上做标注，利用帧间插值处理非关键帧。

## 标注流程

nuScenes 的标注是半自动 + 人工审核的流水线，这是数据工程的重点：

**1. 3D 边框标注（3D Bounding Box）**

- 标注工具：自研的 3D 标注平台，融合 LiDAR 点云和相机图像同时显示
- 标注对象：每个关键帧里所有可见目标
- 属性：位置（x,y,z）、尺寸（长宽高）、朝向（yaw）、速度（通过相邻帧计算）、可见性（0/40/60/80/100%）
- 关键设计：**即使目标只有 0 个 LiDAR 点也要标注**（比如远处的行人），这保证了对稀疏目标的评估公平性

**2. 跨帧追踪（Instance Tracking）**

同一个物体在整个 20 秒场景里保持一致的 instance ID，支持多目标追踪（MOT）任务评估。这需要标注员在时序上保持一致，是标注成本最高的部分。

**3. 语义地图（HD Map）**

4 个地图区域的高精地图，包含：
- 车道线（lane segment）
- 人行横道
- 停车线
- 可行驶区域（drivable area）

地图和传感器数据通过 GPS/IMU 对齐，支持地图辅助感知任务。

**4. 质量控制**

- 标注完成后由独立审核员复查
- 统计每帧的标注密度（判断漏标）
- LiDAR 点云密度低的区域（远处）标注精度较低，论文明确指出这是局限

## 评估指标

nuScenes 针对检测和追踪分别定义了新的指标，修正了 KITTI 指标的一些缺陷：

### nuScenes Detection Score (NDS)

$$\text{NDS} = \frac{1}{10}\left[5\text{mAP} + \sum_{\text{TP} \in \mathcal{TP}} (1 - \min(1, \text{TP}))\right]$$

包含 mAP 和 5 个 True Positive 指标的加权组合：

| TP 指标 | 含义 |
|---------|------|
| ATE（Average Translation Error） | 中心点位置误差（米） |
| ASE（Average Scale Error） | 尺寸误差（1 - IoU） |
| AOE（Average Orientation Error） | 朝向误差（弧度，yaw） |
| AVE（Average Velocity Error） | 速度误差（m/s） |
| AAE（Average Attribute Error） | 属性分类误差 |

**TP 指标详解：什么和什么比，怎么算**

首先明确 GT 是什么：**GT（Ground Truth）是标注员手工标注的真实框**，包含目标的真实位置、尺寸、朝向、速度、属性。模型的预测框（Prediction）和 GT 比较，才能算出误差。

"True Positive"的前提：预测框和某个 GT 框的中心点距离 ≤ 2 米，才算"匹配成功"（TP）。只有 TP 才参与下面 5 个误差的计算；漏检（FN）和误检（FP）会影响 mAP，但不参与 TP 指标。

各指标具体计算：

- **ATE（位置误差）**：预测框中心点 (x, y) 和 GT 框中心点的欧式距离，单位米。z 轴（高度）不参与计算，因为 LiDAR 在高度方向的精度相对可靠，水平位置误差更关键。
  ```
  ATE = sqrt((x_pred - x_gt)² + (y_pred - y_gt)²)
  ```

- **ASE（尺寸误差）**：用 3D IoU 衡量尺寸差异，但去掉了朝向的影响（先把两个框都对齐到同一朝向再算 IoU）。
  ```
  ASE = 1 - IoU_aligned
  ```
  值域 [0, 1]，0 表示尺寸完全一致。

- **AOE（朝向误差）**：预测的 yaw 角（水平朝向角）和 GT yaw 角的差，单位弧度。取最小角度差（因为 180° 对称问题）。
  ```
  AOE = min(|yaw_pred - yaw_gt|, 2π - |yaw_pred - yaw_gt|)
  ```
  对于行人等无固定朝向的类别，AOE 上限为 π（180°）。

- **AVE（速度误差）**：预测的速度向量 (vx, vy) 和 GT 速度向量的 L2 距离，单位 m/s。GT 速度由标注员在相邻关键帧之间计算得到。
  ```
  AVE = sqrt((vx_pred - vx_gt)² + (vy_pred - vy_gt)²)
  ```

- **AAE（属性误差）**：属性是离散分类标签（如车辆状态：行驶中/停止/停放；行人状态：行走/站立/坐着）。AAE = 1 - 分类准确率，值域 [0, 1]。

**NDS 的计算逻辑**：5 个 TP 指标各自先在所有 TP 样本上取平均，然后用 `1 - min(1, error)` 把误差转成"越高越好"的分数，最后和 mAP 一起加权平均：

```
NDS = (1/10) × [5×mAP + (1-ATE) + (1-ASE) + (1-AOE) + (1-AVE) + (1-AAE)]
```

mAP 权重是 5（占一半），5 个 TP 指标各占 1/10。

**为什么不用 KITTI 的 IoU-based mAP**：KITTI 的 2D IoU 对目标朝向不敏感——一辆横向行驶的车和纵向行驶的车，只要框的 2D 投影重合就算对。nuScenes 的 NDS 显式惩罚朝向误差，更符合自动驾驶的实际需求（朝向错误会导致运动预测失败）。

### nuScenes Tracking Score (AMOTA)

Average Multi-Object Tracking Accuracy，在多个召回阈值下平均 MOTAR（Multi-Object Tracking Accuracy with Recall），比传统 MOTA 更对漏检鲁棒。

## 和其他数据集的对比

| 数据集 | 场景数 | LiDAR 线数 | 相机 | 雷达 | 3D 框 | 追踪 ID | 地图 |
|--------|--------|-----------|------|------|-------|---------|------|
| KITTI（2012） | ~200 | 64 线 | 单目（前向） | ✗ | ✓ | ✗ | ✗ |
| Waymo Open（2019） | 1000 | 64 线 | 5（全向） | ✗ | ✓ | ✓ | ✗ |
| **nuScenes（2020）** | **1000** | **32 线** | **6（全向）** | **✓（5个）** | **✓** | **✓** | **✓** |
| Lyft Level 5（2019） | 366 | 40 线 | 7 | ✗ | ✓ | ✗ | ✓ |

nuScenes 的 LiDAR 线数（32 线）低于 Waymo（64 线），是有意为之——模拟量产车上成本可接受的传感器配置，而不是研究用的高端设备。

## 关键设计决策

**1. 两座城市 × 多种天气**

波士顿（有雪、复杂路口）+ 新加坡（热带、左驾、密集行人），覆盖不同驾驶文化和气候条件，减少单一城市的过拟合。

**2. 关键帧采样（2Hz）+ 全帧存储（12Hz 相机 / 20Hz LiDAR）**

只在关键帧标注，但保存所有原始传感器数据。这允许研究者利用非关键帧的时序信息做预测任务，而标注成本只集中在关键帧。

**3. 可见性标注**

每个目标有 0/40/60/80/100% 的可见性标注（表示目标被遮挡的程度）。评估时可以按可见性分组，分析算法在遮挡场景下的表现，而不是把所有目标平等对待。

**4. 速度作为一等公民**

速度是 NDS 的一个独立评估维度，而 KITTI 完全忽略速度。这迫使算法显式建模目标运动，而不只是做静态检测。

## 现状与影响

**定性：数据集本身仍活跃，是学术界 3D 感知的标准基准之一。**

nuScenes 发布后迅速成为 3D 目标检测和多目标追踪的主流 benchmark，截至 2026 年仍是大量论文（PointPillars、CenterPoint、BEVFusion、UniAD 等）的评估基准。

**直接影响的研究方向**：
- **BEV 感知**：nuScenes 的多相机 + LiDAR 配置推动了 BEV（鸟瞰图）感知成为主流，因为全向传感器自然在 BEV 空间融合
- **端到端自动驾驶**：UniAD 等端到端模型都在 nuScenes 上评估，其多任务标注（检测 + 追踪 + 地图）天然支持端到端训练
- **雷达感知复兴**：nuScenes 是少数包含雷达数据的公开数据集，推动了 radar-camera 融合感知研究

**局限性**：
- 32 线 LiDAR 点云稀疏，和现在量产车（通常 64-128 线）有差距
- 1000 个场景总量相比 Waymo（~1000 场景但更多标注帧）或 Argoverse 2 偏少
- 波士顿 + 新加坡的场景分布不覆盖高速公路、乡村道路等场景

**2024–2026 趋势**：nuScenes 作为"入门基准"的地位牢固，但前沿研究开始转向更大规模的数据集（Waymo Open v2、Argoverse 2）或内部私有数据集（Tesla、Waymo 生产数据）。nuScenes 在学术论文里仍普遍使用，主要因为其完整的工具链（devkit）和社区积累。
