# 自动驾驶场景相似性检索

**问题**：有大量自动驾驶录制数据（路口、跟车、变道、行人穿越…），想找"和这个场景相似的历史场景"。能用 embedding 做吗？

**可以，思路完全一样**，只需把"词"换成"场景片段"，把"共现"换成"行为相似"或"感知相似"。原理参见 [Word Embedding](./word-embedding.md)。

## 方案一：用结构化特征训练 embedding（类 word2vec）

把每个场景片段抽成离散特征（障碍物类型、道路结构、天气、速度档位等），拼成一个"场景 token 序列"，然后用 Skip-gram 训练——在同一段行驶序列中相邻出现的场景，视为"共现"。

```
行驶序列：[直道跟车] [弯道加速] [路口等待] [直道跟车] [变道超车]
                        ↑ 窗口内共现 → 向量靠近
```

训练完后，`scene_embedding["路口等待"]` 会和其他路口类场景向量相近。

## 方案二：用连续特征 + 对比学习

如果场景特征是连续的（感知输出的向量、轨迹点等），可以用对比学习（Contrastive Learning）：
- 正例：同一场景的不同时刻 / 数据增强版本 → 向量应相近
- 负例：随机采样的其他场景 → 向量应远离

损失函数和负采样类似，只是正例的定义不同。

## 方案三：直接用感知模型的中间层特征

感知模型（目标检测、BEV 特征等）的中间层输出本身就是一种 embedding——语义相似的场景在特征空间里自然相近，不需要额外训练。直接对特征做 ANN（近似最近邻）检索即可。

## 工程上的关键问题

1. **如何定义一个"场景"**：一帧？一段 3 秒片段？按事件切割？
2. **检索效率**：百亿帧数据，不能暴力遍历。需要 FAISS 等向量检索库，把 embedding 建成索引
3. **相似性的定义**：感知相似（同类障碍物）≠ 行为相似（同样的决策）≠ 难度相似（接管风险）——不同目的需要不同的 embedding 训练方式
