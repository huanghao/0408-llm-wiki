# REINFORCE

## 是什么

REINFORCE（Williams, 1992）是强化学习里最基础的**策略梯度算法**，用于训练面对离散决策的模型。

**核心问题**：语言模型生成 token 是一个离散选择过程，无法直接对"最终结果好不好"做反向传播——因为中间的采样步骤是不可微的。REINFORCE 用一个技巧绕开这个问题：

$$\nabla_\theta \mathbb{E}[R] = \mathbb{E}\left[ R \cdot \nabla_\theta \log p_\theta(a) \right]$$

直觉：如果某个动作 $a$（比如生成某段文字）得到了正奖励 $R$，就**提高**它的生成概率；得到负奖励就**降低**。不需要知道"正确答案是什么"，只需要知道"这次结果好不好"。

**为什么 LLM 预训练可以直接反向传播，而 REINFORCE 要绕一圈？**

关键区别在于"你要优化的目标是否依赖采样"：

- **预训练（NLL）**：目标是"对正确 token 的概率越高越好"。正确 token 是固定已知的（训练数据），不需要采样——直接算 $-\log p(y_t)$，对参数求导，标准反向传播。
- **REINFORCE**：目标是"对最终奖励高的序列，提高其生成概率"。但奖励是在**采样完整序列之后**才能得到——采样本身（argmax 或按概率抽取）是一个离散操作，没有梯度。你没法对"选了哪个 token"这个选择动作直接求导。

REINFORCE 的技巧是绕开采样本身，转而优化"生成这条序列的对数概率"：$\nabla_\theta \log p_\theta(\tau)$，再乘以奖励作为权重。$\log p$ 是可微的，奖励只是一个标量系数，不需要对它求导。

## 和监督学习的区别

| | 监督学习（SFT） | REINFORCE |
|--|---------------|-----------|
| 训练信号 | 每个 token 有正确答案 | 整段输出完成后才有奖励 |
| 梯度来源 | 直接反向传播 | 奖励 × 动作概率梯度 |
| 信号密度 | 密集（每 token 一个梯度） | 稀疏（一段话一个奖励） |
| 需要标注 | 需要（正确输出） | 不需要（只需要能打分的奖励函数） |

## 奖励、Return、Advantage 的区别

批注 #2 和 #3 问到了这三个概念的关系，这里统一说清楚。

**奖励（Reward）$R$**：环境在序列结束时给出的原始信号。在 LLM 场景里，通常是**对整条序列打的一个标量分**——比如数学题答对得 1，答错得 0；奖励模型对整段回答打分。打分的单位是整个序列，不是单个 token。

**Return $G$**：从某一步 $t$ 开始，未来所有奖励的（折扣）累加。在 LLM 的 episode 很短（一次生成就结束）的场景里，每个 token 的 return 通常就等于最终奖励 $R$——因为只有一个奖励时间点。

**Advantage $A$**：Return 减去基线（通常是价值函数的估计）：$A_t = G_t - V(s_t)$。含义是"这一步比预期好多少"，用于降低方差。

**REINFORCE 里用的是哪个？**：最基础的 REINFORCE 直接用 $R$（原始奖励）乘以 $\nabla \log p$。加了基线之后，$R - \bar{R}$ 是一种简化版的 advantage（用批次均值代替价值函数）。PPO 里用的是完整的 advantage（需要 critic 网络估计 $V$）；GRPO 用组内均值作为基线，也是 advantage 的一种近似。

所以：REINFORCE ≈ 用简化 advantage 的策略梯度；PPO = 用精确 advantage（含 critic）的策略梯度。

## 方差问题和基线（Baseline）

REINFORCE 的主要缺点是**方差大**：奖励信号嘈杂，训练不稳定。

常用解法是减去一个**基线**（baseline），通常是同一问题下多次采样的平均奖励：

$$r_i = R_i - \bar{R}$$

这不改变梯度的期望值（无偏），但大幅降低方差——模型学的是"比平均水平好多少"，而不是奖励的绝对值。GRPO（DeepSeek-R1）和 Quiet-STaR 都用了这个技巧。

**奖励如何"乘以"梯度产生更新**：奖励 $R$ 本身是个标量（一个数），$\nabla_\theta \log p_\theta(a)$ 是参数空间里的一个向量（和模型参数同维度）。两者相乘就是把梯度向量按奖励大小缩放——奖励大就用大步长更新，奖励小就用小步长，奖励负就反向更新。本质上还是普通的梯度下降，只是梯度的方向和大小由奖励加权决定，而不是由损失函数直接计算。

## 在 LLM 训练里的应用

| 系统 | 奖励信号 | 备注 |
|------|---------|------|
| STaR / Quiet-STaR | 思考后预测下一词的准确率提升 | 不需要人工标注 |
| GRPO（DeepSeek-R1） | 数学/代码答案对错（0/1） | 见 [RLHF](./rlhf.md) |
| InstructGPT（PPO） | 奖励模型打分 | 见 [RLHF](./rlhf.md) |

REINFORCE 本身太简单、方差太大，工业上很少直接用；PPO、GRPO 都是在它基础上加了方差控制和约束的变体。但理解 REINFORCE 是理解所有策略梯度方法的起点。
