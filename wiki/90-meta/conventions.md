# Knowledge Base Conventions

## Naming

Use lowercase-with-hyphens file names.

Keep dates in `YYYY-MM-DD` form when date-stamping a page.

## Page shape

Prefer:

1. short summary
2. main body
3. links to related pages
4. open questions or next reads

## Index updates

Every durable new page should be added to `wiki/index.md`.

## Log updates

Every meaningful ingest, query filing, or lint pass should append an entry to `wiki/log.md`.

## Source handling

Keep raw materials in `raw/`.

Do not overwrite source files when summarizing them.
